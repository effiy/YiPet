# 编码规范

> JavaScript 代码规范和风格指南

---

## 📋 概述

本文档定义了温柔陪伴助手项目的 JavaScript 编码规范，确保代码风格一致、易于维护。

---

## 🔤 命名规范

### 1. 变量和函数
```javascript
// ✅ 推荐：camelCase
const petManager = {};
function initPetManager() {}

// ❌ 避免：PascalCase（用于类）、snake_case
const PetManager = {};
function init_pet_manager() {}
```

### 2. 常量
```javascript
// ✅ 推荐：UPPER_SNAKE_CASE
const API_ENDPOINTS = {
  SESSIONS: '/api/sessions',
  FAQS: '/api/faqs'
};

// ✅ 组件常量：camelCase 或 UPPER_SNAKE_CASE
const DEFAULT_SIDEBAR_WIDTH = 320;
```

### 3. 文件和目录
```javascript
// ✅ 推荐：kebab-case
chat-window/
├── index.html
└── index.js

// ❌ 避免：camelCase、PascalCase
ChatWindow/
chatWindow/
```

## 🏗️ 代码结构

### 1. 文件组织
```javascript
// ✅ 正确的模块结构
/**
 * 文件头部注释 - 说明文件功能和作者
 */

// 严格模式
'use strict';

// 命名空间和依赖
if (!window.PetManager) window.PetManager = {};
const hooks = window.PetManager.Components.ChatWindowHooks || {};

// 常量定义
const DEFAULT_WIDTH = 850;
const MIN_WIDTH = 600;

// 工具函数
function safeCall(fn, fallback) {
  try {
    return fn();
  } catch (error) {
    return fallback;
  }
}

// 主要功能
class PetManager {
  // 类实现
}
```

### 2. 函数长度
- 函数应保持简短（<= 50行）
- 单一职责原则
- 过长函数应拆分为多个辅助函数

### 3. 缩进
- 使用 2 空格缩进
- 避免使用 tab 或混合缩进
- 保持一致的缩进级别

### 3. 复杂代码注释
```javascript
// 处理流式响应
// 这种方法通过分块接收数据并逐段渲染，
// 提供更好的用户体验
async function handleStreamingResponse(response) {
  // 详细的实现说明
  const reader = response.body.getReader();
  const decoder = new TextDecoder();

  let buffer = '';

  while (true) {
    const { done, value } = await reader.read();

    if (done) break;

    buffer += decoder.decode(value, { stream: true });

    // 渲染累积的内容
    renderChatMessage(buffer);
  }
}
```

## 🎯 代码风格

### 1. 分号使用
```javascript
// ✅ 推荐：使用分号
const x = 5;
const y = x * 2;

function add(a, b) {
  return a + b;
}

// ❌ 避免：省略分号（可能导致隐式分号插入错误）
const x = 5
const y = x * 2

function add(a, b) {
  return a + b
}
```

### 2. 引号使用
```javascript
// ✅ 推荐：单引号字符串
const message = 'Hello world';

// ✅ 模板字符串
const html = `<div class="pet">${petName}</div>`;

// ❌ 避免：双引号（除非符合特定场景）
const message = "Hello world";
```

### 3. 括号和空格
```javascript
// ✅ 推荐：一致的括号和空格使用
if (condition) {
  // 单行条件块
  return result;
}

// ✅ 空格使用
function calculate(x, y) {
  return x * y + 5;
}

const data = {
  key1: value1,
  key2: value2
};

// ❌ 避免：不一致的空格
if(condition){
    return result;
}
```

## 🔧 函数声明

### 1. 函数表达式 vs 声明
```javascript
// ✅ 推荐：函数表达式
const initPetManager = function() {
  // 实现
};

// ✅ 箭头函数（适用于简单函数）
const processMessages = (messages) => {
  return messages.filter(msg => msg.content.length > 0);
};

// ❌ 避免：函数声明（除非在模块作用域顶层）
function initPetManager() {
  // 实现
}
```

### 2. 立即执行函数表达式 (IIFE)
```javascript
// ✅ 正确使用 IIFE 进行封装
(function() {
  'use strict';

  // 模块代码
})();

// ✅ 箭头函数形式
(() => {
  'use strict';

  // 模块代码
})();
```
