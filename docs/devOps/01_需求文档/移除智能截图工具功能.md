# 需求文档：移除智能截图工具功能

## 1. 需求概述

**功能名称**：移除智能截图工具功能

**需求类型**：功能移除

**优先级**：高

**背景**：当前应用支持智能截图工具功能，包括自由选择区域的截图和一键保存。但由于该功能存在以下问题，决定将其从应用中移除：
- 权限要求较高（需要 activeTab 权限）
- 实现复杂，代码维护成本高
- 存在浏览器兼容性问题
- 用户使用频率较低
- 增加了应用的复杂度和加载时间

## 2. 功能分析

### 当前功能实现

#### 2.1 文件结构
```
├── modules/screenshot/content/petManager.screenshot.js     # 截图功能核心实现（863行）
├── modules/extension/background/actions/screenshotHandler.js  # 背景脚本处理
├── assets/styles/content.css                               # 截图功能样式
├── modules/extension/background/bootstrap/imports.js       # 导入声明
└── manifest.json                                           # 权限配置
```

#### 2.2 功能特点
- **自由选择区域**：支持鼠标拖动选择截图区域，双击确认
- **多种截图方法**：使用 Chrome captureVisibleTab API 为主方法，html2canvas 和 getDisplayMedia 为备用方法
- **权限管理**：包含详细的权限诊断和帮助功能
- **预览和保存**：支持截图预览、复制和下载功能
- **可视化反馈**：包含区域选择提示、通知和模态框

#### 2.3 功能流程
1. **启动截图**：点击聊天窗口的截图按钮
2. **权限检查**：检查 Chrome API 可用性和权限
3. **隐藏元素**：隐藏聊天窗口和宠物以获得清晰截图
4. **区域选择**：显示截图背景和选择框，支持鼠标选择
5. **裁剪处理**：根据选择区域裁剪截图
6. **预览保存**：显示预览模态框，支持下载和复制

## 3. 移除范围

### 3.1 需要移除的文件

1. **`modules/screenshot/content/petManager.screenshot.js`** - 截图功能核心实现文件

### 3.2 需要修改的文件

8. **`assets/styles/content.css`** - 移除截图功能相关的 CSS 样式
9. **`docs/核心功能/截取网页区域.md`** - 删除该功能的文档

## 5. 影响分析

### 5.1 功能影响
- 移除后，用户将无法使用智能截图工具
- 聊天窗口中的截图按钮将被移除
- 相关的键盘快捷键（如存在）将失效

### 5.2 性能影响
- 减少了对 Chrome API 的依赖
- 移除了复杂的截图处理逻辑，提高了应用的响应速度
- 减少了应用的加载时间

### 5.3 兼容性影响
- 不再需要 activeTab 权限，降低了权限要求
- 移除了与 Chrome 版本相关的 API 依赖
- 提高了应用在不同浏览器环境中的兼容性

## 6. 替代方案

### 6.1 系统截图工具
用户可以使用操作系统的原生截图功能：
- Windows: Win + Shift + S
- macOS: Cmd + Shift + 4
- Chrome: 可以使用扩展商店中的其他截图工具

### 6.2 复制粘贴
用户可以使用复制功能将内容直接粘贴到聊天窗口：
- 支持文本、图片和其他媒体内容
- 保持了基本的内容分享功能

## 7. 风险评估

### 7.1 低风险
- 功能使用频率较低，用户影响范围有限
- 有明确的替代方案可供使用
- 实施过程简单，代码改动集中

### 7.2 风险缓解
- 在功能移除前，确保所有相关代码已正确清理
- 更新文档，明确告知用户功能已移除
- 确保其他功能的稳定性不受影响
