# 架构设计

> 温柔陪伴助手 - 项目架构和关键模块设计

---

## 📋 架构概述

温柔陪伴助手是一个基于 Chrome Manifest V3 的浏览器扩展，采用模块化架构设计，将功能分解为独立的模块，便于维护和扩展。

---

## 🏗️ 整体架构

### 1. 三层架构模式

#### ✨ 内容脚本层 (Content Scripts)
- **位置**：`modules/extension/content-scripts/` 和 `modules/pet/content/`
- **作用**：在网页上下文中运行的脚本
- **主要组件**：`PetManager` - 宠物管理核心类
- **特点**：直接操作 DOM，处理与网页的交互

#### 🔄 后台服务层 (Background Service)
- **位置**：`modules/extension/background/`
- **作用**：在后台持续运行的服务 worker
- **主要功能**：消息路由、权限管理、状态持久化
- **特点**：无 DOM 访问权限，处理系统级任务

#### 🪟 弹出界面层 (Popup)
- **位置**：`modules/extension/popup/`
- **作用**：扩展图标点击后的弹出界面
- **主要功能**：快速操作、设置入口
- **特点**：简单轻量，与后台服务通信

## 🔄 通信机制

### 1. Chrome 运行时通信

#### 消息类型
```javascript
// 内容脚本 → 后台
{
  action: 'pet:toggle',
  data: { visible: true }
}

// 后台 → 内容脚本
{
  action: 'pet:toggle:response',
  data: { success: true, message: '宠物已显示' }
}
```

#### 通信流程
```
网页 ←→ Content Script ←→ Background Service ←→ Popup
```

### 1. PetManager 核心管理类

**位置**：`modules/pet/content/`

```javascript
// PetManager 核心结构
class PetManager {
  constructor(options) {
    this.config = options.config
    this.state = {}
    this.components = {}
  }

  // 初始化方法
  async init() {
    await this.loadConfig()
    await this.setupComponents()
    await this.setupEventListeners()
  }

  // 功能方法
  async toggleVisible() {
    // 切换宠物可见性
  }

  async sendMessage(message) {
    // 发送消息到 AI
  }
}
```

### 2. API 层设计

**位置**：`core/api/`

#### ApiManager 类
- 统一的请求和响应处理
- 请求拦截和响应拦截
- 错误处理和重试机制

#### 服务类
- `SessionService` - 会话管理服务
- `FaqService` - FAQ 管理服务

### 3. 存储管理

**位置**：`core/utils/storage/`

#### Chrome Storage
- `petGlobalState` - 宠物全局状态
- `petChatWindowState` - 聊天窗口状态

## 🚀 启动流程

### 1. 扩展加载流程
1. `manifest.json` - 扩展配置加载
2. `content-scripts` - 内容脚本注入
3. `bootstrap.js` - 启动引导
4. `PetManager` - 初始化

### 2. 初始化序列
```javascript
// modules/extension/content-scripts/content.js
chrome.runtime.onMessage.addListener((message) => {
  if (message.action === 'pet:init') {
    initPetManager()
  }
})

// core/bootstrap/bootstrap.js
async function initPetManager() {
  const config = await loadConfig()
  const petManager = new PetManager({ config })
  await petManager.init()

  window.PetManager = petManager
}
```
