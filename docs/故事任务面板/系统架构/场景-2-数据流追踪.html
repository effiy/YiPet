YiPet Data Flow Architecture

请求链路 · 事件流 · 存储路径 · 关键数据流可追踪到端点

AI Chat Flow

  • User message → ChatInput (Vue) → PetManager dispatch
  • ai.prompt.js constructs system prompt + conversation context
  • ai.api.js calls ApiManager → TokenManager → RequestClient
  • 3 retries with exponential backoff, X-Token auto-injected
  • Streaming response rendered via marked.js → ChatMessages

Storage Paths

  • chrome.storage.local — all persistent state
  • pet_global_state — pet position, visibility, settings
  • pet_chat_window_state — open/close, last active session
  • storage.onChanged — cross-context sync (SW + CS)
  • LRU eviction when quota exceeded (petOssFiles first)

Message Routing

  • 7 action categories → 6 handler mapping in MessageRouter
  • chrome.tabs.sendMessage — bidirectional SW/CS communication
  • WeChat Work bot integration via WeWorkHandler
  • Popup → Service Worker → Content Script relay chain
  • Extension install/update events → ExtensionHandler
