## 功能概述

🎯 **提升可维护性**：将超 2000 行的模块拆分为职责单一的小文件
⚡ **保持功能稳定**：通过兼容层确保所有现有功能继续正常工作
📖 **统一拆分策略**：沿用已验证的渐进式重构方法论

## 验收标准

### P0 - 必须通过

- **目标模块拆分完整**：成功拆分 petManager.editor.js、petManager.mermaid.js、petManager.ai.js

- **注释完善**：关键逻辑有清晰的注释说明

## 功能详情

### 编辑器模块拆分

**功能说明**：将 petManager.editor.js（2154 行）拆分为核心逻辑和 UI 交互模块
**价值**：编辑器功能与 UI 展示分离，便于独立维护和测试
**解决的痛点**：单个文件包含所有编辑器相关代码，职责不清

### Mermaid 模块拆分

**功能说明**：将 petManager.mermaid.js（1871 行）拆分为渲染器和交互模块
**价值**：Mermaid 核心渲染逻辑与 UI 交互分离
**解决的痛点**：CDN 加载、渲染逻辑、预览功能混杂在同一文件中

### AI 对话模块拆分

**功能说明**：将 petManager.ai.js（1608 行）拆分为 API 封装和提示词管理
**价值**：AI API 调用逻辑与提示词业务逻辑分离
**解决的痛点**：重试机制、token 管理、角色配置混杂在一起

### 加载顺序优化

**功能说明**：更新 manifest.json 中的 content_scripts 加载顺序
**价值**：确保新模块先加载，兼容层后加载
**解决的痛点**：依赖关系导致初始化错误

## 使用场景示例

### 📋 场景 1：编辑器模块拆分

**背景**：petManager.editor.js 包含上下文编辑器、会话信息编辑器、Markdown 预览等所有功能
**操作**：
1. 拆分为 petManager.editor.core.js（编辑器核心逻辑）
2. 拆分为 petManager.editor.ui.js（UI 交互和事件处理）
3. 保留原文件作为兼容层
**结果**：代码职责清晰，后续可独立优化渲染或交互逻辑

### 🎨 场景 2：Mermaid 模块拆分

**背景**：petManager.mermaid.js 包含 CDN 加载、图表渲染、预览交互等所有功能
**操作**：
1. 拆分为 petManager.mermaid.renderer.js（渲染核心和 CDN 加载）
2. 拆分为 petManager.mermaid.ui.js（UI 交互和预览功能）
3. 保留原文件作为兼容层
**结果**：渲染逻辑稳定，UI 交互可独立迭代
