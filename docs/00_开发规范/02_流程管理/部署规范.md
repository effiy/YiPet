# 部署规范

> 扩展打包和部署流程

---

## 📋 概述

本文档定义了温柔陪伴助手 Chrome 扩展的打包、测试和部署流程。

---

## 🚀 打包流程

### 1. 打包前检查
- 确保所有代码已提交到 Git
- 运行代码审查和静态分析
- 检查所有功能是否正常工作
- 更新版本号

### 2. 版本号管理
- 遵循语义化版本规范 (SemVer)
- 格式：主版本号.次版本号.修订号
- 示例：1.1.1

### 3. 打包步骤
```bash
# 1. 更新 manifest.json 中的版本号
"version": "1.1.1"

# 2. 清理临时文件
# 删除 .DS_Store、.git 目录等

# 3. 创建 zip 包
zip -r gentle-companion-1.1.1.zip . \
  -x "*.git*" \
  -x "docs/*" \
  -x "node_modules/*" \
  -x ".DS_Store"
```

---

## 🧪 发布前测试

### 1. 功能测试
- 加载扩展到 Chrome
- 测试所有核心功能
- 验证权限配置
- 测试不同网页环境

### 2. 兼容性测试
- 测试 Chrome 最新版本
- 测试 Chrome 稳定版本
- 测试不同操作系统
- 测试不同屏幕分辨率

---

## 📦 发布流程

### 1. Chrome Web Store 发布
- 登录 Chrome Web Store 开发者控制台
- 上传新的 zip 包
- 更新应用描述和截图
- 提交审核

### 2. 内部发布
- 在 GitHub Releases 中发布
- 上传 zip 包
- 添加发布说明
- 标记版本号
