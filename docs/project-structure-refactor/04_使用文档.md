## 功能介绍

本项目对 YiPet 温柔陪伴助手的目录结构进行了模块化重构，将原本分散的 `core/`、`libs/`、`assets/` 合并到 `cdn/` 目录下，将 `modules/pet/content/` 拆分为 core/modules/features 三层结构，将 `modules/extension/` 提升为顶级 `extension/` 目录。重构后的目录结构对齐 YiWeb 代码结构规范，更适合模块化开发。

- 🎯 **目录职责清晰**：每个目录名准确反映其职责，新开发者无需猜测
- ⚡ **分层架构明确**：cdn（共享资源）+ modules（业务逻辑）+ extension（扩展系统）三大块
- 🔧 **开发效率提升**：文件位置可预测，修改影响范围可控

## 快速开始

### 前置条件

- [x] 理解 YiWeb 代码结构规范（`rules/代码结构.md`）
- [x] 了解 Chrome Extension Manifest V3 的工作方式
- [x] 已完成目录结构重构（代码变更已提交）

### 30秒上手

1. 查看顶层目录：`cdn/`（共享资源）、`modules/`（业务模块）、`extension/`（扩展系统）
2. 修改共享工具 → 编辑 `cdn/utils/` 下的文件
3. 修改业务逻辑 → 编辑 `modules/pet/core/`、`modules/pet/modules/`、`modules/pet/features/` 对应文件
4. 修改扩展系统 → 编辑 `extension/` 下的文件
5. 修改 UI 组件 → 编辑 `cdn/components/` 下的文件
6. 🎉 重构后目录结构清晰，文件位置一目了然！

## 操作场景

### 📋 场景1：查找共享工具函数

**适用情况**：需要使用或修改项目中共享的工具函数

**操作步骤**：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 进入 `cdn/utils/` | 所有共享工具函数在此 |
| 2 | 按功能分类选择子目录 | api/、dom/、error/、logging/、media/、messaging/、runtime/、session/、storage/、time/、ui/ |
| 3 | 编辑对应文件 | 每个文件职责单一 |

**预期结果**：快速定位到目标工具函数，修改只影响该工具的职责范围

**注意事项**：✅ `cdn/utils/` 中的函数被多个模块依赖，修改前需检查影响范围

### 🎨 场景2：添加新的宠物角色

**适用情况**：需要为新角色添加图片资源和配置

**操作步骤**：

1. 在 `cdn/assets/images/` 下创建英文名子目录（如 `programmer/`）
2. 在新目录中添加 `icon.png`（角色图标）和 `run/` 子目录（动画帧）
3. 在 `modules/pet/modules/petManager.roles.js` 中添加角色配置
4. 在 `cdn/core/config.js` 的 `DEFAULTS.PET_ROLE` 中确认默认角色名

**预期结果**：新角色图片和配置在预期位置，角色切换功能可正常使用新角色

**注意事项**：❌ 目录名必须使用英文小写+短横线，禁止中文目录名

### 📋 场景3：修改聊天窗口功能

**适用情况**：需要修改聊天相关的逻辑或 UI

**操作步骤**：

| 步骤 | 目录 | 说明 |
|------|------|------|
| 1 | `cdn/components/chat/` | 修改聊天 UI 组件 |
| 2 | `modules/pet/features/petManager.chat*.js` | 修改聊天逻辑 |
| 3 | `modules/pet/features/petManager.message.js` | 修改消息处理逻辑 |

**预期结果**：核心逻辑和 UI 代码分离清晰，修改不互相影响

**注意事项**：✅ UI 修改在 `cdn/components/`，逻辑修改在 `modules/pet/features/`

### 🎨 场景4：调试消息路由问题

**适用情况**：消息无法从 Popup 正确传递到 Content Script

**操作步骤**：

1. 在 `extension/messaging/` 查看 `messageRouter.js` 的路由规则
2. 在 `extension/background/actions/` 查看各处理器（petHandler、screenshotHandler 等）
3. 在 `extension/background/services/` 查看 `injectionService.js` 和 `tabMessaging.js`

**预期结果**：消息路由和处理器的组织清晰，快速定位问题

**注意事项**：✅ 消息路由已从 background 子目录提升到与 background 同级

### 📋 场景5：更新第三方库版本

**适用情况**：需要升级 Vue.js、marked.js 等第三方库

**操作步骤**：

1. 在 `cdn/libs/` 中找到目标库文件（如 `vue.global.js`）
2. 替换为新版本文件（保持文件名不变）
3. 验证依赖该库的模块功能正常

**预期结果**：所有依赖该库的模块自动使用新版本

**注意事项**：❌ 不要修改文件名，`manifest.json` 和 `injectionService.js` 通过文件名引用

## 常见问题

### 💡 基础问题

**Q1：重构后为什么看不到 `core/`、`libs/`、`assets/` 目录？**

这些目录已合并到 `cdn/` 目录下。`core/` → `cdn/core/`，`libs/` → `cdn/libs/`，`assets/` → `cdn/assets/`，`core/utils/` → `cdn/utils/`（提升一级）。

**Q2：`modules/pet/content/` 目录去哪了？**

`content/` 中间层已去掉。文件按职责分为 `modules/pet/core/`（核心实现）、`modules/pet/modules/`（功能模块）、`modules/pet/features/`（特性文件）。

**Q3：为什么角色图片目录从中文改成了英文？**

中文目录名（医生、教师、甜品师、警察）存在跨平台编码风险。已改为 doctor、teacher、chef、police。`config.js` 中 `DEFAULTS.PET_ROLE` 也从 `'教师'` 改为 `'teacher'`。

**Q4：`modules/extension/` 去哪了？**

已提升为顶级 `extension/` 目录。background/、popup/、messaging/ 现在与 modules/ 同级。

**Q5：修改路径后扩展加载报错怎么办？**

检查 `manifest.json`、`imports.js`、`injectionService.js` 三处路径是否一致。这三处是路径引用的"铁三角"，必须同步更新。

### ⚙️ 高级问题

**Q6：如何确认所有路径引用已更新？**

搜索 `chrome.runtime.getURL(` 调用，确认所有动态加载路径已更新。检查 Chrome DevTools 控制台是否还有 404 错误。

**Q7：新增功能模块应该放在哪里？**

- 核心类定义 → `modules/pet/core/`
- 功能扩展模块 → `modules/pet/modules/`
- 特性文件 → `modules/pet/features/`
- Vue UI 组件 → `cdn/components/`
- 共享工具 → `cdn/utils/`

**Q8：`injectionService.js` 和 `manifest.json` 的路径列表为什么必须一致？**

`manifest.json` 定义 Chrome 自动注入的脚本，`injectionService.js` 定义手动注入的脚本（用于动态注入到新打开的标签页）。两者是镜像副本，不一致会导致 `PetManager` 等全局对象未定义。

### 🔧 故障排查

**Q9：扩展加载后宠物不显示？**

检查 `cdn/core/config.js` → `modules/pet/core/petManager.core.js` 的加载链是否完整。查看控制台是否有 `PetManager is not defined` 错误。

**Q10：角色切换后图片不显示？**

确认角色目录名使用英文（doctor/teacher/chef/police），确认 `imageResourceManager.js` 中路径前缀为 `cdn/assets/images/`，确认 `chrome.runtime.getURL()` 参数正确。

## 技巧与提示

### 💡 实用技巧

1. **路径三处同步**：修改任何文件路径后，同步更新 manifest.json、imports.js、injectionService.js
2. **英文命名**：所有目录名使用英文小写+短横线，避免中文路径
3. **分层定位**：core=核心、modules=功能模块、features=特性，按职责找文件
4. **共享组件**：Vue 组件在 `cdn/components/`，不在业务模块内
5. **独立扩展**：扩展系统在 `extension/`，与业务模块分离
