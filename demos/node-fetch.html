node-fetch 使用事例

概述

node-fetch v3.3.2 不直接在测试中 import。它作为 Response 接口规范参考 —— mock 工厂函数遵循其形状，确保测试 mock 与真实 fetch 行为一致。

Mock 工厂函数

三个核心 mock 工厂，遵循 node-fetch Response 接口形状。

makeApiResponse — 成功响应

export function makeApiResponse(data, status = 200) {
  return new Response(JSON.stringify({ code: 0, msg: 'ok', data }), {
    status,
    headers: { 'Content-Type': 'application/json' }
  });
}

makeApiError — 业务错误

export function makeApiError(code, msg, httpStatus = 200) {
  return new Response(JSON.stringify({ code, msg }), {
    status: httpStatus,
    headers: { 'Content-Type': 'application/json' }
  });
}

mockFetchError — 网络异常

export function mockFetchError() {
  return new TypeError('Failed to fetch');
}

请求验证模式

通过 vi.fn() 的 mock.calls 验证请求参数。

// 发送请求后验证 fetch 调用参数
const [url, init] = globalThis.fetch.mock.calls[0];
expect(init.method).toBe('GET');
expect(init.headers['X-Token']).toBe('test-token');
expect(url).toContain('/api/sessions');

// 验证请求体
const body = JSON.parse(init.body);
expect(body.page).toBe(1);

三工厂使用模式

正常响应

fetch
  .mockResolvedValueOnce(
    makeApiResponse({ items: [] })
  );

业务错误

fetch
  .mockResolvedValueOnce(
    makeApiError(401, '未授权')
  );

网络错误

fetch
  .mockRejectedValueOnce(
    mockFetchError()
  );

重试链路

fetch
  .mockRejectedValueOnce(mockFetchError())
  .mockResolvedValueOnce(
    makeApiResponse({ retry: 'ok' })
  );
