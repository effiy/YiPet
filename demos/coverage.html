@vitest/coverage-v8 使用事例

概述

@vitest/coverage-v8 v3.1.1 基于 V8 原生覆盖率引擎。配置 provider: 'v8'，输出 text/json/html 三种报告。

vitest 配置

import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    coverage: {
      provider: 'v8',
      include: ['core/**/*.js', 'modules/**/*.js'],
      exclude: ['**/node_modules/**'],
      reporter: ['text', 'json', 'html'],
      thresholds: {
        statements: 60,
        branches: 50,
        functions: 55,
        lines: 60,
      },
    },
  },
});

三种报告格式

text — 终端输出

# npm run test:coverage
File                     | % Stmts | % Branch | % Funcs | % Lines
-------------------------|---------|----------|---------|--------
All files                |   72.34 |    58.21 |   65.10 |   73.45
 core/config.js          |   95.00 |    80.00 |  100.00 |   95.00
 core/utils/api/request  |   68.42 |    45.83 |   55.00 |   70.15
 modules/pet/content/    |   61.33 |    42.50 |   48.00 |   62.80

html — 可视化报告

运行 test:coverage 后，在 coverage/ 目录查看交互式 HTML 报告：行级高亮（绿=覆盖/红=未覆盖）、文件浏览器、覆盖率过滤。

逐场景覆盖率目标

不同测试场景的覆盖率目标不同，按风险分级。

场景1 核心逻辑
82%
场景2 API 接口
75%
场景3 数据持久化
100%
场景4 异常边界
91%
场景5 集成回归
68%

CI/CD 门禁

# package.json
{
  "scripts": {
    "test": "vitest run",
    "test:coverage": "vitest run --coverage"
  }
}

# CI 门禁: 覆盖率低于阈值 → 构建失败
npx vitest run --coverage
# exit code 1 if thresholds not met
