## 一、实施概览

### 1.1 实施目标

本次重构的目标是消除项目中的代码坏味道，包括：

- **重复类**: 合并重复的 ErrorHandler 和 LoggerUtils 实现
- **过度细分**: 简化 pet 模块中过于细分的文件结构
- **命名不一致**: 统一所有文件使用 kebab-case 命名
- **中文目录**: 将中文目录名改为英文
- **死代码**: 移除废弃的 screenshot 模块

### 1.2 实施范围

本次变更涉及：
- **删除文件**: 4 个（core/utils/api/logger.js, core/utils/api/error.js, modules/screenshot/）
- **重命名文件**: 64 个（统一 kebab-case）
- **修改文件**: 6 个（manifest.json, injection-service.js 等）
- **新增文件**: 0 个

## 五、风险与缓解

### 5.1 已识别风险

| 风险 | 严重程度 | 缓解措施 |
|------|----------|----------|
| 文件重命名后引用遗漏 | 高 | 已仔细比对 manifest.json 和 injection-service.js，保持完全一致 |
| 中文角色配置可能兼容性问题 | 中 | 已更新默认角色为英文，但用户自定义配置需向后兼容处理 |
| Git 历史追踪可能受影响 | 低 | 使用 `git mv` 确保文件历史连续性 |

### 5.2 后续建议

1. **测试验证**: 建议在多个浏览器环境下测试所有功能
2. **回退方案**: 如遇问题，可通过 `git reset --hard HEAD~1` 快速回退
3. **文档更新**: 更新项目开发文档中的文件路径引用
