# 日志规范

> 日志记录和管理

---

## 📋 概述

本文档定义了温柔陪伴助手项目的日志规范，确保日志能够帮助开发者调试和诊断问题。

---

## 🎯 日志级别

### 1. DEBUG - 调试信息
```javascript
// 用于开发调试，详细的执行流程
Logger.debug('进入 initPetManager 函数');
Logger.debug('加载配置文件', config);
```

### 2. INFO - 一般信息
```javascript
// 重要的业务流程信息
Logger.info('聊天窗口已打开');
Logger.info('会话创建成功', sessionId);
```

### 3. WARN - 警告信息
```javascript
// 潜在的问题，但不影响正常使用
Logger.warn('API响应时间过长', { duration: 5000 });
Logger.warn('使用了默认配置');
```

### 4. ERROR - 错误信息
```javascript
// 错误发生，功能可能受影响
Logger.error('API请求失败', error);
Logger.error('无法加载组件', { component: 'ChatWindow' });
```

---

## 🛠️ 日志实现

### 1. Logger 类
```javascript
const Logger = {
  debug: (msg, data) => {
    if (config.DEBUG) {
      console.log(`[DEBUG] ${msg}`, data || '');
    }
  },
  info: (msg, data) => {
    console.log(`[INFO] ${msg}`, data || '');
  },
  warn: (msg, data) => {
    console.warn(`[WARN] ${msg}`, data || '');
  },
  error: (msg, data) => {
    console.error(`[ERROR] ${msg}`, data || '');
  }
};
```

### 2. 日志格式
```javascript
// ✅ 推荐的日志格式
[2026-03-18 14:30:25] [DEBUG] 初始化 PetManager
[2026-03-18 14:30:26] [INFO] 聊天窗口已打开
[2026-03-18 14:30:27] [WARN] API响应时间过长 { duration: 3000 }
[2026-03-18 14:30:28] [ERROR] 请求失败 { url: '/api/sessions', error: 'Network Error' }
```

## 🔒 日志安全

### 1. 避免记录敏感信息
```javascript
// ❌ 错误做法
Logger.debug('用户设置', {
  apiToken: 'sk-xxx-yyy-zzz', // 敏感信息
  username: 'user@example.com'
});

// ✅ 正确做法
Logger.debug('用户设置', {
  apiToken: '***已隐藏***',
  username: 'user@example.com'
});
```
