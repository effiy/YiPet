## 🌍 环境配置

### 环境概述

项目支持三种运行环境，每种环境对应不同的 API 端点和配置：

| 环境 | 环境标识 | API 端点 | 说明 |
|------|----------|----------|------|
| 生产环境 | `production` | `https://api.effiy.cn` | 默认环境，正式使用 |
| 开发环境 | `development` | `http://localhost:8000` | 本地开发调试 |
| 测试环境 | `staging` | `https://test-api.effiy.cn` | 测试验证 |

### 切换环境

#### 方法一：全局变量设置

在加载配置前设置全局变量：

```javascript
// 在浏览器控制台或代码中执行
window.__PET_ENV_MODE__ = 'development';
```

**注意**：必须在扩展加载配置之前设置，否则不会生效。建议在 `core/config.js` 加载前执行。

#### 方法二：修改配置文件

直接修改 `core/config.js` 中的默认环境：

```javascript
// core/config.js
const DEFAULT_ENV = 'development'; // 修改为想要的环境
```

### 环境检测

扩展会按以下顺序检测当前环境：

1. `window.__PET_ENV_MODE__` 全局变量
2. `core/utils/env/envUtils.js` 中的检测逻辑
3. 默认回退到 `production`

## 🔌 API 端点配置

### 配置文件位置

- **主配置文件**：`core/config.js`
- **端点常量**：`core/constants/endpoints.js`
- **环境工具**：`core/utils/env/envUtils.js`

### 配置结构

#### core/config.js

```javascript
const ENV_CONFIG = {
  production: {
    API_BASE_URL: 'https://api.effiy.cn',
    DEBUG: false,
    // 其他生产环境配置
  },
  development: {
    API_BASE_URL: 'http://localhost:8000',
    DEBUG: true,
    // 其他开发环境配置
  },
  staging: {
    API_BASE_URL: 'https://test-api.effiy.cn',
    DEBUG: true,
    // 其他测试环境配置
  }
};
```

#### core/constants/endpoints.js

```javascript
export const API_ENDPOINTS = {
  SESSIONS: '/api/v1/sessions',
  FAQS: '/api/v1/faqs',
  AI_CHAT: '/api/v1/chat',
  // 其他端点
};
```

### 添加新端点

1. 在 `core/constants/endpoints.js` 中添加端点常量
2. 在对应 API 服务中使用该端点

```javascript
// core/constants/endpoints.js
export const API_ENDPOINTS = {
  // ... 现有端点
  NEW_FEATURE: '/api/v1/new-feature',
};

// 使用方式
import { API_ENDPOINTS } from '../../constants/endpoints.js';
const url = config.API_BASE_URL + API_ENDPOINTS.NEW_FEATURE;
```

### 自定义 API 端点

如果需要使用自定义的 API 服务器：

1. 复制 `core/config.js` 中的现有环境配置
2. 修改 `API_BASE_URL` 为你的服务器地址
3. 切换到对应的环境

## 💾 Chrome 存储

### 存储键说明

扩展使用 Chrome `storage.local` API 存储数据，主要键值如下：

| 键名 | 数据类型 | 说明 |
|------|----------|------|
| `petGlobalState` | Object | 宠物全局状态（可见性、位置、大小、颜色） |
| `petChatWindowState` | Object | 聊天窗口状态（位置、大小） |
| `petSettings` | Object | 用户设置（API 令牌、AI 配置） |
| `petDevMode` | Boolean | 开发模式标志 |
| `petSessions` | Array | 会话列表 |
| `petFaqs` | Array | FAQ 列表 |

### petGlobalState 结构

```javascript
{
  visible: true,              // 宠物是否可见
  position: { x: 100, y: 100 }, // 宠物位置
  size: { width: 80, height: 80 }, // 宠物大小
  color: '#ffb6c1'           // 主题颜色
}
```

### petChatWindowState 结构

```javascript
{
  position: { x: 200, y: 200 }, // 窗口位置
  size: { width: 400, height: 600 } // 窗口大小
}
```

### petSettings 结构

```javascript
{
  apiToken: 'sk-xxx',         // API 令牌
  aiModel: 'gpt-4',           // AI 模型
  aiTemperature: 0.7,          // 温度参数
  currentRole: 'teacher',      // 当前角色
  autoSave: true              // 自动保存
}
```

### 存储操作示例

#### 读取存储

```javascript
chrome.storage.local.get(['petSettings'], (result) => {
  const settings = result.petSettings || {};
  console.log(settings);
});
```

#### 写入存储

```javascript
chrome.storage.local.set({
  petSettings: {
    apiToken: 'new-token',
    aiModel: 'gpt-4'
  }
}, () => {
  console.log('设置已保存');
});
```

#### 监听存储变化

```javascript
chrome.storage.onChanged.addListener((changes, namespace) => {
  if (namespace === 'local' && changes.petSettings) {
    console.log('设置已更新', changes.petSettings.newValue);
  }
});
```

## ⚙️ 功能配置

### AI 配置

#### 配置项说明

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `apiToken` | String | `''` | API 访问令牌 |
| `aiModel` | String | `'gpt-4'` | AI 模型名称 |
| `aiTemperature` | Number | `0.7` | 生成温度 (0-1) |
| `maxTokens` | Number | `2000` | 最大令牌数 |
| `systemPrompt` | String | `''` | 系统提示词 |

#### 修改 AI 配置

方式一：通过设置界面
1. 打开聊天窗口
2. 点击设置按钮
3. 在 AI 设置中修改配置

方式二：通过代码
```javascript
PetManager.updateSettings({
  apiToken: 'your-token',
  aiModel: 'gpt-4',
  aiTemperature: 0.8
});
```

### 宠物角色配置

#### 角色列表

| 角色标识 | 名称 | 描述 |
|----------|------|------|
| `teacher` | 教师 | 知识渊博的学习伙伴 |
| `doctor` | 医生 | 关心健康的医疗顾问 |
| `pastryChef` | 甜品师 | 甜蜜温馨的生活伴侣 |

#### 切换角色

方式一：通过设置界面
1. 打开聊天窗口
2. 点击角色选择


### 快捷键配置

#### 默认快捷键

| 功能 | Windows/Linux | Mac |
|------|--------------|-----|
| 切换宠物显示/隐藏 | `Ctrl+Shift+P` | `Cmd+Shift+P` |
| 打开/关闭聊天窗口 | `Ctrl+Shift+X` | `Cmd+Shift+X` |

#### 修改快捷键

修改 `manifest.json` 中的 `commands` 配置：

```json
{
  "commands": {
    "toggle-pet": {
      "suggested_key": {
        "default": "Ctrl+Shift+P",
        "mac": "Command+Shift+P"
      },
      "description": "切换宠物显示"
    },
    "open-chat": {
      "suggested_key": {
        "default": "Ctrl+Shift+X",
        "mac": "Command+Shift+X"
      },
      "description": "打开聊天窗口"
    }
  }
}
```

**注意**：修改快捷键后需要重新加载扩展。

## 🔧 高级配置

### 调试配置

#### 启用调试模式

```javascript
// 方法一：设置全局变量
window.__PET_ENV_MODE__ = 'development';

// 方法二：设置开发模式标志
chrome.storage.local.set({ petDevMode: true });
```

#### 调试日志

调试模式下会输出详细的日志信息：

```javascript
// core/utils/api/logger.js
const Logger = {
  log: (msg, ...args) => {
    if (config.DEBUG) {
      console.log(`[Pet] ${msg}`, ...args);
    }
  },
  error: (msg, ...args) => {
    console.error(`[Pet Error] ${msg}`, ...args);
  }
};
```

### 权限配置

在 `manifest.json` 中配置扩展权限：

```json
{
  "permissions": [
    "storage",
    "activeTab",
    "scripting",
    "tabs"
  ],
  "host_permissions": [
    "<all_urls>"
  ]
}
```

#### 权限说明

| 权限 | 说明 |
|------|------|
| `storage` | 访问 Chrome 存储 |
| `activeTab` | 访问当前标签页 |
| `scripting` | 注入脚本到页面 |
| `tabs` | 访问标签页信息 |
| `<all_urls>` | 访问所有网站 |

## ❓ 常见问题

### Q: 如何确认当前环境？

A: 在浏览器控制台执行：
```javascript
console.log(window.__PET_ENV_MODE__);
// 或查看扩展日志中的环境信息
```

### Q: 修改配置后不生效？

A: 请尝试以下步骤：
1. 重新加载扩展（chrome://extensions/ → 刷新图标）
2. 刷新网页
3. 检查浏览器控制台是否有错误

### Q: 如何重置所有配置？

A: 在浏览器控制台执行：
```javascript
chrome.storage.local.clear(() => {
  console.log('所有配置已重置');
});
```

### Q: API Token 安全吗？

A: API Token 仅存储在本地 Chrome 存储中，不会上传到任何第三方服务器。
