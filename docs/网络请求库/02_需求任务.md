## 主要操作场景

### 主要操作场景: RequestClient 基础请求

**关联需求文档**: [需求文档-RequestClient](./01_需求文档.md#requestclient---http-请求客户端)
**验证工具建议**: verification-loop
**前置条件**:
1. Chrome 扩展环境可用
2. manifest.json 已正确配置脚本加载顺序

**操作步骤**:
1. 创建 RequestClient 实例（可选配置超时、模式等）
2. 调用 get/post/put/delete 方法发送请求
3. 处理返回数据或捕获错误
4. 可选：使用 abortKey 取消请求

**预期结果**:
1. 请求成功发送并返回解析后的数据
2. 错误情况下抛出分类后的异常
3. 超时控制生效
4. 请求可被取消

**验证关注点**:
- Fetch API 调用正确
- URL 参数构建正确
- 请求头设置正确
- 响应解析逻辑（JSON/Text/Blob）正确
- 超时和取消功能正常
