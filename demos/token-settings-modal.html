TokenSettingsModal 使用事例

概述

TokenSettingsModal 是 Token 认证设置弹窗，167 行代码。提供 Token 输入（密码模式）、模型下拉选择、输入验证、Enter/Escape 键盘快捷键、保存/取消操作。支持模板和 h() 双模式渲染。

状态管理

const store = reactive({
  token: '',          // 当前输入的 Token
  model: '',          // 当前选择的模型
  invalid: false,      // Token 是否无效（空）
  models: [],         // 可用模型列表（从 manager 获取）
})

// 初始化: 读取当前 Token 和模型
store.token = manager.getCurrentToken() || ''
store.model = manager.currentModel || ''
store.models = manager.getAvailableModels() || []

// 键盘快捷键
Enter → save()
Escape → cancel()

方法详解

save — 验证 + 持久化

save() {
  if (!store.token || !store.token.trim()) {
    store.invalid = true
    return   // Token 为空，标记无效，不关闭
  }

  store.invalid = false
  const token = store.token.trim()

  // 通过 ApiManager 保存 Token
  manager.saveApiToken(token)

  // 更新当前模型
  if (store.model) manager.currentModel = store.model

  // 关闭弹窗
  manager.closeTokenSettingsModal()
}

Token 调用链

三级降级: 环境变量 → chrome.storage → 空

// TokenManager.getToken() 读取链路
getToken() {
  // L1: 环境变量 (CI/CD 注入)
  if (typeof process !== 'undefined' && process.env?.API_X_TOKEN)
    return process.env.API_X_TOKEN

  // L2: chrome.storage.local (用户手动设置)
  const result = await chrome.storage.local.get(['apiToken'])
  return result.apiToken || null

  // L3: 空 → 提示用户设置 Token（打开 TokenSettingsModal）
}
