## 功能概述

YiPet 温柔陪伴助手当前目录结构存在命名不一致、层级混乱、职责交叉等问题，影响开发效率与代码可维护性。本次重构旨在将项目目录结构按模块化开发需求重新组织，与 YiWeb 代码结构规范（`rules/代码结构.md`）对齐，同时保证 Chrome Extension（Manifest V3）的功能完整性。

- 🎯 **统一命名规范**：消除 `core/` 与 `cdn/` 的概念混淆，使目录名准确反映职责
- ⚡ **清晰分层架构**：将资源、核心工具、功能模块、扩展系统分离到独立层级，降低耦合
- 📖 **提升可维护性**：每个目录的职责明确，新开发者能快速定位文件位置

## 验收标准

### P0 - 必须通过

- [ ] 所有目录名使用英文小写 + 短横线分隔，不含中文或大写字母
- [ ] `manifest.json` 中所有脚本和资源路径可正确解析
- [ ] 扩展加载后全部功能正常运行（宠物显示、聊天、截图、会话管理、FAQ）
- [ ] 功能模块按 core/modules/features 三层结构组织
- [ ] Vue 组件目录与逻辑代码目录分离
- [ ] `injectionService.js` 中 CONTENT_SCRIPT_FILES 路径同步更新

### P1 - 应该通过

- [ ] 公共样式按功能分类存放（base/components/features）
- [ ] 第三方库集中在 `cdn/libs/` 目录
- [ ] 扩展系统代码（background/popup/messaging）独立组织
- [ ] 角色图片目录使用英文名（doctor/teacher/chef/police）
- [ ] 所有文件路径引用更新后无控制台 404 错误

### P2 - 可以有

- [ ] `docs/` 目录按规范文档体系组织
- [ ] `docs/structure.md` 同步更新
- [ ] `.gitignore` 排除构建产物和临时文件

## 功能详情

### 统一命名与资源合并

**功能说明**：将 `core/`、`libs/`、`assets/` 三个顶级目录合并到 `cdn/` 目录下按子职责分层。

**价值**：与 YiWeb 项目体系一致，新开发者无需猜测目录含义

**解决的痛点**：`core/utils/api/` 与 `core/api/` 职责重叠；`libs/` 命名与 YiWeb 规范不一致；`docs/structure.md` 中已规划 `cdn/` 但实际不存在

### 功能模块分层

**功能说明**：将 `modules/pet/content/` 下混合放置的 14+ 个文件按职责拆分为 `core/modules/features` 三个子目录，去掉 `content/` 中间层。

**价值**：每个文件的职责一目了然，新增功能时知道该放在哪里

**解决的痛点**：无法一眼区分核心代码和功能扩展

### 扩展系统独立

**功能说明**：将 `modules/extension/` 提升为顶级 `extension/` 目录，将消息路由从 background 子目录提升到与 background 同级。

**价值**：扩展系统代码与业务逻辑解耦，便于独立测试和维护

**解决的痛点**：消息处理器和服务的组织方式不一致，`integrations/wework/` 嵌套较深

### 中文目录英文化

**功能说明**：将角色图片目录从中文名改为英文名，消除跨平台兼容性风险。

**价值**：路径在所有环境中稳定可解析

**解决的痛点**：中文路径在某些系统和工具中可能导致编码问题

## 使用场景示例

#### 📋 场景1：新开发者首次查看项目

> **背景**：新加入的开发者需要理解项目结构
>
> **操作**：查看 `cdn/` 了解共享资源、`modules/` 了解业务模块、`extension/` 了解扩展系统
>
> **结果**：目录名清晰反映职责，3 分钟内找到目标文件

#### 🎨 场景2：添加新的宠物角色

> **背景**：需要添加"程序员"角色
>
> **操作**：在 `cdn/assets/images/programmer/` 添加角色图标，在角色配置模块添加新角色
>
> **结果**：角色图片和配置都在预期位置，命名规范一致

#### 📋 场景3：修复聊天功能的 Bug

> **背景**：聊天窗口出现渲染问题
>
> **操作**：在 `modules/pet/features/` 定位聊天特性逻辑，在 `cdn/components/chat/` 定位 UI 组件
>
> **结果**：核心逻辑和 UI 代码分离清晰，修改不互相影响
