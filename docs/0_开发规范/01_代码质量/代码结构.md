# 代码结构规范

> 温柔陪伴助手 - 项目代码组织和架构规范

---

## 概述

本规范定义了温柔陪伴助手项目的代码组织和架构设计原则，确保项目结构清晰、模块划分合理、易于维护和扩展。

---

## 目录结构原则

### 1. 按功能模块划分
- 所有功能模块统一放在 `modules/` 目录下
- 每个功能模块有独立的目录结构
- 模块内部按功能进一步细分

### 2. 核心系统分离
- 核心系统模块放在 `core/` 目录下
- 包括配置、启动、常量、API 层等基础功能
- 核心模块尽量保持稳定，避免频繁修改

### 3. 资源文件独立
- 所有静态资源放在 `assets/` 目录下
- 按类型分类：styles、images、icons 等
- 第三方库统一放在 `libs/` 目录下

---

## 模块结构规范

### 功能模块目录结构
```
modules/
└── {moduleName}/
    ├── components/       # Vue 组件（如适用）
    ├── content/          # 核心逻辑实现
    │   ├── core/         # 核心实现
    │   └── modules/      # 功能子模块
    ├── styles/           # 样式文件
    └── ...
```

### 核心模块目录结构
```
core/
├── config.js           # 集中式配置
├── bootstrap/          # 启动/初始化代码
├── constants/          # 常量定义
├── api/                # API 集成层
│   ├── core/           # API 管理器
│   ├── services/       # API 服务
│   └── utils/          # API 工具
└── utils/              # 全局工具模块
```

## 文件命名规范

### JavaScript 文件
- 使用小写字母和连字符：`pet-manager.js`
- 特殊模块使用驼峰：`petManager.js`
- 常量文件使用大写：`CONSTANTS.js`

### 样式文件
- 使用小写字母和连字符：`pet-style.css`
- 与对应 JS 文件同名

### 组件文件
- Vue 组件使用大驼峰：`ChatWindow.vue`
- 组件目录使用小驼峰：`chatWindow/`

---

## 代码组织原则

### 1. 单一职责
- 每个文件只负责一个功能
- 函数尽量保持短小精悍
- 模块内部高内聚，模块之间低耦合

### 2. 依赖管理
- 明确模块之间的依赖关系
- 避免循环依赖
- 使用依赖注入降低耦合

### 3. 可测试性
- 代码结构便于单元测试
- 依赖可 mock
- 功能模块化便于测试

### 业务逻辑层
- PetManager 核心逻辑
- 功能模块实现
- 状态管理

### 数据层
- API 集成层
- 存储管理
- 数据处理

## 最佳实践

1. **先设计后编码**：在编码前先设计模块结构
2. **及时重构**：发现结构问题及时重构
3. **保持一致性**：遵循统一的代码结构规范
4. **文档更新**：结构变更时及时更新文档
