YiPet Dependency Impact Analysis

Dependency tiers, loading order, and failure blast radius analysis

Dependency Tiers

  • • P0: Chrome Extension Runtime — heartbeat dependency
  • • P1: API Backend — chat/session/FAQ all depend on it
  • • P2: CDN assets — UI degradation only
  • • Impact: P0>P1>P2 triage before any upgrade

Loading Order

  • • manifest.json — permission gate (required first)
  • • Service Worker — background messaging hub
  • • content_scripts — PetManager.init() triggers
  • • Vue CDN — ChatWindow mounts after load

Module Fanout

  • • config.js: Root dependency, 0 upstream deps
  • • PetManager.core: State hub, orchestrates 9+ modules
  • • ChatWindow: Vue root, fans out to 4+ sub-components
  • • Service Worker <-> CS via chrome.tabs IPC
