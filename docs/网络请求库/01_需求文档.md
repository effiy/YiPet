## 功能概述

YiPet 网络请求库是一个轻量级、功能完整的 HTTP 请求库，专为 Chrome 扩展环境设计。它提供了统一的请求接口、Token 管理、错误处理、日志记录和服务封装等功能，支持零构建直接运行。

🎯 **核心价值**: 提供可靠、易用、可扩展的 HTTP 请求基础能力
⚡ **零构建运行**: 基于原生 JavaScript 和 Fetch API，无需构建工具
📖 **完整文档**: 提供详细的使用文档和示例

## 验收标准

### P0 - 必须通过

- **核心请求功能完整**: RequestClient 支持 GET/POST/PUT/DELETE 方法
- **Token 管理可用**: TokenManager 支持获取、保存、清除 Token
- **错误处理完善**: 支持网络错误、超时错误、认证错误等分类处理
- **向后兼容**: 使用 IIFE 模式挂载到全局，不依赖 ES Modules
- **零构建运行**: 代码可直接在 Chrome 扩展中运行

### P1 - 应该通过

- **拦截器机制可用**: 支持请求和响应拦截器链
- **重试机制有效**: 网络错误和超时错误可自动重试
- **日志记录完整**: 请求、响应、错误都有日志记录
- **服务层封装完整**: SessionService 和 FaqService 提供完整 CRUD 操作

### P2 - 可以有

- **请求取消功能**: 支持通过 AbortController 取消请求
- **批量处理优化**: SessionService 支持批量保存队列
- **统计信息收集**: 提供请求成功率等统计数据

## 功能详情

### RequestClient - HTTP 请求客户端

**功能说明**: 基于 Fetch API 的轻量级 HTTP 客户端，提供统一的请求接口、超时控制、取消请求、自动重试等功能。
**价值**: 简化 HTTP 请求代码，统一请求处理逻辑
**解决的痛点**: 避免重复编写 Fetch 请求代码，统一错误处理方式

### TokenManager - Token 管理器

**功能说明**: 管理 API 认证 Token，支持环境变量注入、Chrome 存储持久化、同步/异步获取等多种方式。
**价值**: 安全、便捷地管理认证信息
**解决的痛点**: Token 分散管理、获取方式不统一的问题

### ErrorHandler - 错误处理器

**功能说明**: 统一错误分类、处理和重试机制，提供网络错误、超时错误、认证错误等分类处理。
**价值**: 统一错误处理逻辑，提高代码可维护性
**解决的痛点**: 错误处理分散、错误类型不明确的问题

### Logger - 日志管理器

**功能说明**: 统一的日志记录工具，支持日志级别控制、请求/响应日志格式化输出。
**价值**: 便于调试和问题定位
**解决的痛点**: 日志输出不统一、调试困难的问题

### ApiManager - API 管理器基类

**功能说明**: 服务层基类，整合请求客户端、Token 管理、错误处理、日志记录，提供拦截器机制。
**价值**: 统一业务服务层的基础能力
**解决的痛点**: 各业务服务重复实现基础功能的问题

### SessionService - 会话服务

**功能说明**: 封装会话相关 API，提供会话 CRUD、搜索、批量操作、队列保存等功能。
**价值**: 简化会话管理业务代码
**解决的痛点**: 会话操作分散、批量处理复杂的问题

### FaqService - FAQ 服务

**功能说明**: 封装常见问题相关 API，提供 FAQ CRUD、标签筛选、搜索、批量排序等功能。
**价值**: 简化 FAQ 管理业务代码
**解决的痛点**: FAQ 操作分散、排序逻辑复杂的问题

## 使用场景示例

### 📋 场景 1: 发送简单 GET 请求

**背景**: 需要从 API 获取数据列表
**操作**:
```javascript
const client = new RequestClient();
const data = await client.get('https://api.example.com/items', { limit: 10 });
```
**结果**: 返回解析后的 JSON 数据

### 📋 场景 2: 发送带 Token 的 POST 请求

**背景**: 需要提交数据到需要认证的 API
**操作**:
```javascript
const api = new ApiManager('https://api.example.com');
const result = await api.post('/items', { name: 'Test' });
```
**结果**: Token 自动添加到请求头，返回操作结果

### 🎨 场景 3: 使用 SessionService 管理会话

**背景**: 需要保存用户会话数据
**操作**:
```javascript
const service = new SessionService('https://api.example.com');
await service.saveSession({ key: 'session_1', title: 'My Session' });
```
**结果**: 会话数据自动保存到服务器
