## 功能概述

YiPet 温柔陪伴助手当前目录结构存在命名不一致、层级混乱、职责交叉等问题，影响开发效率与代码可维护性。本次重构旨在将项目目录结构按模块化开发需求重新组织，消除 `core/`/`libs/`/`assets/` 三个顶级目录的职责分散，将 `modules/pet/content/` 下 14+ 混合文件按 core/modules/features 三层结构重组，将 Vue 组件从 `modules/pet/components/` 提升到共享层，将 `modules/extension/` 独立为顶级 `extension/` 目录，同时保证 Chrome Extension（Manifest V3）的路径依赖完整。

- 🎯 **统一命名规范**：消除 `core/` 与 `cdn/` 的概念混淆，使目录名准确反映职责
- ⚡ **清晰分层架构**：将资源、核心工具、功能模块、扩展系统分离到独立层级，降低耦合
- 📖 **提升可维护性**：每个目录的职责明确，新开发者能快速定位文件位置

## 验收标准

### P0 - 必须通过

- 所有目录名使用英文小写+短横线分隔，不含中文或大写字母
- `manifest.json` 中所有脚本和资源路径可正确解析
- 扩展加载后全部功能正常运行（宠物显示、聊天、截图、会话管理、FAQ）
- 功能模块按 core/modules/features 三层结构组织
- Vue 组件目录与逻辑代码目录分离
- `injectionService.js` 中 CONTENT_SCRIPT_FILES 路径同步更新

### P1 - 应该通过

- 公共样式按功能分类存放（base/components/features）
- 第三方库集中在 `cdn/libs/` 目录
- 扩展系统代码（background/popup/messaging）独立组织
- 消息路由器处理器按功能分类存放
- 角色图片目录使用英文名（doctor/teacher/chef/police）
- 所有文件路径引用更新后无控制台 404 错误

### P2 - 可以有

- `docs/` 目录按规范文档体系组织
- 添加 `.gitignore` 排除构建产物和临时文件
- `docs/structure.md` 同步更新

## 功能详情

### 统一命名与资源合并

当前 `core/`、`libs/`、`assets/` 三个顶级目录职责分散。按 YiWeb 代码结构规范，应合并为 `cdn/` 目录下按子职责分层。

- **价值**：与 YiWeb 项目体系一致，新开发者无需猜测目录含义
- **解决的痛点**：`core/utils/api/` 与 `core/api/` 职责重叠；`libs/` 命名与 YiWeb 规范不一致；`docs/structure.md` 中已规划 `cdn/` 但实际不存在

### 功能模块分层

当前 `modules/pet/content/` 下混合了 14+ 个文件，核心实现（`petManager.core.js`）、功能模块（`modules/petManager.*.js`）、特性文件（`petManager.chat.js` 等）缺乏分层。

- **价值**：每个文件的职责一目了然，新增功能时知道该放在哪里
- **解决的痛点**：无法一眼区分核心代码和功能扩展

### 扩展系统独立

当前 `modules/extension/` 混合在业务模块中，应独立为顶级目录。

- **价值**：扩展系统代码与业务逻辑解耦，便于独立测试和维护
- **解决的痛点**：消息处理器和服务的组织方式不一致，`integrations/wework/` 嵌套较深

## 使用场景示例

📋 **场景 1：新开发者首次浏览项目**

- **背景**：新加入团队的开发者需要了解项目结构
- **操作**：浏览 `cdn/` 目录下的分层结构，快速定位配置、工具、组件位置
- **结果**：无需询问即可理解各目录职责，5 分钟内找到目标文件

📋 **场景 2：添加新的宠物功能模块**

- **背景**：需要为宠物添加新的交互功能
- **操作**：在 `modules/pet/modules/` 下创建新模块文件，按命名规范 `petManager.{feature}.js`
- **结果**：新模块自动遵循 core/modules/features 三层结构，职责清晰

📋 **场景 3：扩展后台消息处理**

- **背景**：需要为后台添加新的消息处理器
- **操作**：在 `extension/background/actions/` 下添加处理器，在 `extension/messaging/` 注册路由
- **结果**：处理器独立于业务模块，路由集中管理
