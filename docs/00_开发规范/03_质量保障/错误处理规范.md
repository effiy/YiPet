# 错误处理规范

> 异常处理和错误报告

---

## 📋 概述

本文档定义了温柔陪伴助手项目的错误处理规范，确保异常能够被正确捕获、处理和报告。

---

## 🎯 错误处理原则

### 1. 防御性编程
- 假设外部调用可能失败
- 验证所有输入数据
- 提供默认值和回退方案

### 2. 不要静默失败
- 捕获异常但不要忽略
- 记录所有重要错误
- 提供有意义的错误消息

---

## 🛡️ 错误捕获

### 1. Try-Catch 块
```javascript
// ✅ 推荐：明确的错误处理
try {
  const response = await fetch(url);
  const data = await response.json();
  return data;
} catch (error) {
  Logger.error('API请求失败', error);
  // 提供回退方案
  return fallbackData;
}

// ❌ 避免：空的 catch 块
try {
  // 代码
} catch (e) {
  // 什么都不做 - 静默失败
}
```

### 2. Promise 错误处理
```javascript
// ✅ 推荐：使用 .catch() 处理 Promise 错误
apiCall()
  .then(data => processData(data))
  .catch(error => {
    Logger.error('操作失败', error);
    showErrorToUser(error.message);
  });

// ✅ 推荐：结合 async/await 和 try-catch
async function fetchData() {
  try {
    const result = await apiCall();
    return result;
  } catch (error) {
    Logger.error('获取数据失败', error);
    throw error; // 重新抛出让调用者处理
  }
}
```

## 📝 错误消息

### 1. 用户友好的消息
```javascript
// ✅ 推荐：用户友好的错误消息
function showError(error) {
  const userMessage = {
    'NETWORK_ERROR': '网络连接失败，请检查网络后重试',
    'AUTH_ERROR': 'API 令牌无效，请重新设置',
    'TIMEOUT_ERROR': '请求超时，请稍后重试'
  };

  const message = userMessage[error.code] || '发生未知错误，请重试';
  showNotification(message, 'error');
}

// ❌ 避免：显示技术性错误信息给用户
showNotification(error.stack, 'error');
```

### 2. 详细的技术日志
```javascript
// ✅ 推荐：记录详细的技术信息用于调试
Logger.error('API请求失败', {
  url: url,
  status: response.status,
  error: error.message,
  stack: error.stack
});
```

## 🔄 错误恢复

### 1. 自动重试
```javascript
// ✅ 推荐：对可恢复的错误实现重试逻辑
async function fetchWithRetry(url, options = {}, maxRetries = 3) {
  let lastError;

  for (let i = 0; i < maxRetries; i++) {
    try {
      const response = await fetch(url, options);
      if (!response.ok) throw new Error(`HTTP ${response.status}`);
      return await response.json();
    } catch (error) {
      lastError = error;
      Logger.warn(`请求失败，第 ${i + 1} 次重试`);
      await sleep(1000 * (i + 1)); // 指数退避
    }
  }

  throw lastError;
}
```
