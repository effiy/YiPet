## 2. 设计背景

### 2.1 当前问题

当前聊天记录导出为图片功能存在以下问题：

1. **性能问题**：
   - 使用 html2canvas 库进行 DOM 转图片操作，性能较低
   - 导出大段聊天记录时，会导致页面卡顿
   - 高清图片生成过程消耗较多内存和 CPU 资源

2. **维护成本**：
   - 依赖外部库 html2canvas (340KB)，增加了包体积
   - 需要维护复杂的样式复制和图片加载逻辑
   - 导出功能代码量较大（483 行）

3. **用户体验**：
   - 导出过程需要等待，用户反馈体验不佳
   - 生成的图片在不同设备上显示效果不一致
   - 导出功能的使用频率较低

### 2.2 需求分析

- **功能目标**：移除聊天记录导出为图片功能
- **用户需求**：保留其他导出方式（会话导入/导出、复制功能）
- **技术需求**：简化架构，提高应用性能
