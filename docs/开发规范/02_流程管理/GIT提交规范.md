# GIT 提交规范

> 提交信息规范和工作流程

---

## 📋 概述

本文档定义了温柔陪伴助手项目的 Git 提交规范，确保提交历史清晰、可维护。

---

## 🎯 提交信息格式

### 1. 基本格式
```
<类型>: <主题>

<详细说明>

<关联Issue>
```

### 2. 提交类型
```
feat: 新功能
fix: 修复bug
refactor: 代码重构
docs: 文档更新
style: 代码格式调整
test: 测试相关
chore: 构建或工具变更
```

### 3. 示例
```
feat: 添加宠物角色切换功能

实现了宠物角色的切换功能，支持教师、医生、甜品师和警察四种角色。
- 在聊天窗口添加角色选择按钮
- 创建角色配置模块
- 实现角色状态管理

#123
```

---

## 🔧 工作流程

### 1. 分支管理
```
main               # 主分支（生产环境）
develop            # 开发分支
feature/xxx        # 功能分支
hotfix/xxx         # 热修复分支
```

### 2. 提交规范检查
- 使用 Husky 进行提交规范检查
- 配置 commitlint 规则
- 提交前运行测试
